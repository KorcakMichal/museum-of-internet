# Museum of Internet

> A browser game where you can live in the real internet — content generated by AI.

## Architecture

The project follows a **modular architecture** so that each concern can be developed and swapped out independently.

```
museum-of-internet/
├── index.html            # HTML entry point
├── vite.config.js        # Vite build config
├── package.json
├── public/               # Static assets (images, sounds, …)
└── src/
    ├── main.js           # Bootstrap: wires all modules together
    ├── game/
    │   ├── index.js      # Public API re-exports
    │   ├── engine.js     # Game loop & module orchestration
    │   ├── world.js      # World / scene management (consumes AI content)
    │   └── player.js     # Player state & movement
    ├── ai/
    │   ├── index.js      # Public API re-exports
    │   └── contentGenerator.js  # AI content generation (LLM integration point)
    ├── ui/
    │   ├── index.js      # Public API re-exports
    │   └── hud.js        # HUD / scene rendering
    └── utils/
        └── index.js      # Shared helpers (EventEmitter, clamp, sleep, …)
```

### Key design decisions

| Concern | Module | Notes |
|---------|--------|-------|
| Game loop | `src/game/engine.js` | Runs `requestAnimationFrame`; calls `update` on world & player each tick |
| AI content | `src/ai/contentGenerator.js` | Single `callAI` stub — replace with any LLM API (OpenAI, Anthropic, …) |
| World state | `src/game/world.js` | Fetches AI-generated scene on init; holds entities |
| UI / HUD | `src/ui/hud.js` | Pure DOM rendering, no framework dependency |
| Utilities | `src/utils/index.js` | `EventEmitter`, `clamp`, `sleep` |

## Getting started

```bash
npm install
npm run dev      # start dev server (hot-reload)
npm run build    # production build → dist/
npm run preview  # preview production build
npm test         # run tests
```

## Adding AI content

1. Open `src/ai/contentGenerator.js`
2. Replace the `callAI` stub with your preferred LLM provider
3. Extend `generateScene`, `generateNarrative`, or `generateItem` as needed
